This page is a classroom-focused workflow for using Avida-ED with minimal setup time.

## Before class (15–30 minutes)

1. **Choose version:** start with [Choose a Version](../get-started/choose-version.md).
2. **Test the classroom setup** on the same device type your students will use.
3. Decide what students will produce:
   - a short written explanation
   - a table of results
   - exported data (CSV) + graph
4. Prepare a one-page “what to submit” instruction (especially if using an LMS).

## In class (core sequence)

### 1) Frame the scientific question

Use a specific, testable question such as:

- How does changing the mutation rate affect adaptation?
- How does changing the environment affect which traits persist?

### 2) Establish a baseline run

- Use default settings.
- Have students record what they observe and the key settings used.

### 3) Change one variable

- Change one condition only.
- Run again for the same amount of time/steps.
- Compare outcomes across groups.

### 4) Require an explanation

A minimal, high-quality student response includes:

- what changed (the independent variable)
- what was measured/observed (dependent variable)
- what pattern occurred
- a plausible explanation supported by observations

## Assessment options

- Short exit ticket (5 minutes): “What changed, what happened, why?”
- CER format: Claim–Evidence–Reasoning
- Group comparison: reconcile conflicting results and discuss variability

## Common pitfalls

- Students change multiple variables at once.
- Students describe results without explaining mechanisms.
- Students treat a single run as “proof” rather than evidence.
