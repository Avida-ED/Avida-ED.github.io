title: Accessibility
---

Avida-ED aims to support accessible classroom use for all learners.

## Accessibility practices on this site

- Clear headings and consistent navigation
- Descriptive link text (not “click here”)
- Alt text for meaningful images
- Keyboard-accessible navigation
- Visible focus indicators
- Underlined text links so links are not identified by color alone
- Captions and transcripts for videos before they are treated as primary learning resources
- HTML-first lesson pages for core teaching paths where possible
- Legacy PDFs and media clearly labeled as archival/supporting resources

## Current limitations and how the site handles them

- Some legacy curriculum materials still exist only as PDFs. Where that is true, the site now points users to HTML-first lesson pages when available and labels the PDF as a supporting file.
- Some legacy video resources exist on external platforms and may not yet meet the accessibility standard for captions and transcripts. Those resources are preserved as references, but they should not be the only path for completing core tasks.
- Draft or oversized local video files should not be published as required course content until captions, transcript coverage, and a stable hosting plan are in place.

## If you encounter a barrier

If you encounter an accessibility barrier:

1. Note the page URL.
2. Describe what you tried to do and what happened.
3. Include your browser and device type.
4. If possible, include a screenshot.

Submit the report through:

- [Avida-ED website issues](https://github.com/Avida-ED/Avida-ED.github.io/issues)
- your course instructor, if you need an immediate accommodation for class use

When you report a barrier, include what alternative format or workflow would let you continue your task.

## Alternative formats

If a classroom activity relies on a file format that is not accessible for a learner, instructors can request an alternative format or use the HTML equivalent pages when available.
