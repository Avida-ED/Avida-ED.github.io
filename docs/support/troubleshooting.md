---
title: Troubleshooting
---

Use this checklist when Avida-ED does not launch or when assignment workflow
steps fail.

## If the app does not launch

- confirm you started from the current [Launch Avida-ED](/app) page
- reload the page once before changing settings
- try a current desktop browser if you were using an older or mobile setup
- if you are in a computer lab or on a managed school device, confirm the
  browser is not locked to an older unsupported version
- check whether privacy tools or pop-up blocking interfere with the app
- if a class depends on a local fallback, verify that the offline or legacy
  setup instructions were followed exactly using
  [Download and fallback guidance](/download)

## If a classroom activity behaves unexpectedly

- confirm all students are using the same version
- make sure students are not changing more than one variable at a time
- rerun the same setup once to see whether the pattern is reproducible
- compare the assignment against the current
  [Curriculum](/curriculum), [Lab Manual](/curriculum/lab-manual), or
  [Model Lessons](/curriculum/model-lessons) page if students are using older
  handouts

## If students cannot save or export files

- confirm the device permits downloads
- check whether the browser blocks downloads by default
- verify that the file appears in the expected download location

## If a student is following an older video or handout

- confirm which Avida-ED version the material was written for
- check whether the page is describing older browser beta behavior or a
  standalone app workflow
- redirect the student to [For Students](../students/index.md) for current
  launch and workflow guidance
- use transcript pages when a video is helpful but the interface shown is not an
  exact match for the current app

## If you must use a local Windows standalone setup

The legacy download instructions for Windows followed this pattern:

1. Run the standalone installer locally.
2. Allow the local server to run on private networks if prompted.
3. Open the browser to the local `localhost` address provided by the package.
4. Reuse the local server executable for later launches.

Those instructions should be treated as legacy fallback guidance rather than the
mainline student path.

## When to report an issue

Report the problem when:

- the issue persists after a reload
- it affects multiple users on the same setup
- it prevents completion of a lesson or assignment

Include the details listed on the [Support](./index.md) page.
For a clean report format, use [Reporting Problems](./reporting-problems.md).
