---
title: Troubleshooting Checklist
---

Use this page when Avida-ED does not launch correctly or behaves unexpectedly during class.

## Quick checks (most common)

1. **Reload the page** once.
2. Confirm you are using a **modern browser** (see [System Requirements](./system-requirements.md)).
3. Try a **private/incognito window** to rule out cached scripts or extensions.
4. If your school uses content filters, confirm the Avida-ED domain is allowed.

## If the app launches but controls do not respond

- Try keyboard navigation (Tab/Shift+Tab/Enter) to confirm focus is moving.
- Disable browser extensions for the site (privacy blockers can interfere).
- Try a second browser (Chrome ↔ Firefox) to isolate a browser-specific issue.

## If students cannot save or export files

- Confirm the device and account permissions allow downloads.
- Check if the LMS or browser blocks downloads by default.
- Use a simple file naming convention (e.g., `lastname_avidaed_run1.csv`) to prevent overwrite confusion.

## If the classroom network is the issue

- Test on a different network (hotspot) to confirm.
- Provide the IT department with:
  - the full launch URL
  - the time and device type tested
  - any error messages

## Report a problem

If you can reproduce an issue reliably, capture:

- browser name + version
- device type (Chromebook, Windows laptop, iPad, etc.)
- the exact step when failure occurs
- screenshot (if permitted)

Then send it via the Support page.
