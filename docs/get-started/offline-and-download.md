---
title: Offline and Download Options
---

Most classes should use the web-based version of Avida-ED, but the legacy site also maintained download and offline options for instructors working in constrained environments.

## Recommended current order

1. Use **Avida-ED 4** in the browser when possible.
2. Use legacy **Avida-ED 3** only when existing materials require it.
3. Use a download/offline path only when internet access or classroom network policy makes the web version impractical.

## Avida-ED-Eco (v4) Anti-BitRot Edition

The legacy download page identified **Avida-ED-Eco (v4) Anti-BitRot Edition** as the long-term-use version and pointed instructors to the `aed-docker` GitHub repository for installation and use instructions.

This is the best candidate for an offline or long-term preservation path if the project still intends to maintain it.

## Legacy standalone Avida-ED 3.3.5

The old download page also provided standalone legacy packages for use where there was no internet access.

### Mac

Legacy instructions said to:

- download and unzip the package
- move `Avida-ED Web.app` into `Applications`
- approve opening it because it comes from an unknown source

### Windows

Legacy instructions said to:

- run the installer locally
- allow the bundled local web server on private networks
- open `http://localhost:8000/AvidaEd.html`
- relaunch `winserver.exe` for future use

## Migration note

These standalone instructions were written for legacy releases. Keep them available for archive/support reasons, but do not present them as the default classroom path unless they are actively maintained and tested.
